
cart process (not the REST API process):
user chooses that they want an account.
cart sets username to "CREATE" and generates time-stamped password 
integration process calls SP IntegrationAddOrderAccount 
SP IntegrationAddOrderAccount generates initial/default username based on new CustomerID and it 
 also writes a record in the SystemIntegrationUserEmailQueue table
a batch process reads the SystemIntegrationUserEmailQueue every X min. (5, I think), sends
an email notice with the credentials, and then removes the entry from the SystemIntegrationUserEmailQueue table

REST API process:
jquery form posts a UserRequest object to 
www.asecurecart.net/api/User
Model calls SPs UsernameExists and APICreateUserAccount
